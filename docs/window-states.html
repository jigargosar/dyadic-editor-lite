The two rules everything follows

  1. Hiding is always minimize, then hide. Minimize first hands focus back to whatever app you were using; hide then drops the window to the tray. A bare hide is never used.
  2. Showing is always followed by focus. Restore it if minimized, then show it, then focus it — focus comes last so it isn't dropped.

Not confirmed yet

  • assumptionOn Windows, show() is what brings the window to the front — so focus must come after show/restore or it gets dropped.
  • assumptionThe native minimize can't be canceled, so we route it straight to hide instead of preventing it.
  • riskThat Shift+Space re-focuses reliably on every cycle — the "visible but unfocused → front" path is the main worry.
